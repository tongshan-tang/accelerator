# 赛题二：一种稀疏矩阵运算的芯片系统的设计与实现

## 1. 赛题背景
毫米波雷达通过发射波长为毫米级的电磁波，并接收目标反射回来的回波信号，从而计算出目标的距离、速度和角度信息。在毫米波雷达的信号处理以及点云跟踪过程中，存在大量稀疏矩阵的运算需求。例如，天线采用不等间距布阵会引入稀疏矩阵运算，而目标点云的离散分布也会带来稀疏矩阵的处理任务。随着毫米波雷达探测精度和分辨率的不断提升，所涉及的稀疏矩阵往往具有非常大的行数和列数。因此，如何在有限的计算资源下高效地实现大尺寸稀疏矩阵的运算，已成为毫米波雷达信号处理与目标跟踪中的关键技术之一。

## 2. 赛题及相关描述

### 2.1 相关概念定义
- **稀疏矩阵**：非 0 元素占比在 30% 以下。

- **矩阵行重**：定义为每一行非 0 元素的个数，占比为 30% 以下。

- **矩阵列重**：定义为每一列非 0 元素的个数，占比为 30% 以下。

- **矩阵乘法定义**：C = A * B，A 为 M 行 K 列矩阵，B 为 K 行 N 列矩阵，得到结果为 M 行 N 列矩阵。

- **矩阵加法和减法定义**：C = A + B 和 C = A - B，A 为 M 行 N 列矩阵，B 为 M 行 N 列矩阵，得到结果为 M 行 N 列矩阵。

### 2.2 设计任务
设计一个高效实现稀疏矩阵**乘法、加法、减法**的硬件加速系统。

### 2.3 赛题详细要求

#### A. 硬件加速器实现：矩阵乘法（C = A * B）
用一套硬件加速器实现矩阵 C = A * B 的功能。其中 A 的尺寸为 M 行 K 列，B 的尺寸为 K 行 N 列，且 A 和 B 均具有稀疏特性：

1. **A 具有最大行重的稀疏特性**，即每一行最大的非 0 元素个数为 NUM = Floor(K * 30%)。

2. **B 具有最大列重的稀疏特性**，即每一列最大的非 0 元素个数为 NUM = Floor(K * 30%)。

3. **参数配置**：A 和 B 的尺寸 M, N, K 等系数均可支持软件配置，定义 M, N, K 的区间为 [2^4, 2^9]。

#### B. 硬件加速器实现：矩阵加减法（C = A + B 和 C = A - B）
用一套硬件加速器实现矩阵 C = A + B 和 C = A - B 的功能。其中 A 的尺寸为 M 行 N 列，B 的尺寸为 M 行 N 列，且 A 和 B 也均具有稀疏特性（矩阵加减法的 A 和 B 稀疏特性与乘法略有不同）：

1. **A 和 B 均具有最大行重**的稀疏特性，即每一行最大的非 0 元素个数为 NUM = Floor(N * 30%)。

2. **A 和 B 无列重特性**。

#### C. 数据存储要求
- **输入数据**：在矩阵运算时，输入 A 和 B 以一定格式存储在 FPGA 的 SRAM（可充分利用 FPGA 的 BRAM 或者寄存器资源实现）中。存储的 A 和 B 格式可以自定义，且 A 和 B 分别在 2 个独立的 bank 中。

- **输出数据**：矩阵运算结果 C 以一定格式存储在 FPGA 的 SRAM（可充分利用 FPGA 的 BRAM 或者寄存器资源实现）中。存储的 C 格式可以自定义，且 C 存储在一个不同于 A 和 B 的独立 bank 中。

- **数据格式规格**：矩阵元素定义为标准的 FL16 (IEEE754)。在乘法运算中，存储单个 A 矩阵 SRAM 的 size 不超过 `M * K * 2` Bytes，B 的 size 不超过 `N * K * 2` Bytes。在加减运算中 A 和 B 矩阵 SRAM 的 size 不超过 `M * N * 2` Bytes。

#### D. 实现平台
用 FPGA 实现上述硬件加速计算功能，建议 FPGA 型号为 Xilinx **Kintex-7** 系列及以上平台。

评判标准
a)性能：设计硬件加速器处理矩阵乘，加，减运算最低性能要求为75GB/s（处理带宽=A矩阵Rawsize/处理时间），所设计加速器方案处理带宽基于75GB/s每提升10%，该项加一个单位的基准分，每低于10%，该项减去一个单位的基准分。

b)面积：设计FPGAvivadocheck无timing问题，占用LUT，Flip-Flop，BRAM，时钟，复位及布线资源占比合理，整体资源使用率低于75%（Kintex-7）--（该项会根据不同的FPGA系列资源分布进行归一化后评分）。

c)功耗：整体信号翻转率低于60%。

d)逻辑资源利用率：乘法器，加法器，减法器在不同矩阵和运算法则的平均利用率为不低于70%，利用率每提升5%，该项评分加一个单位的基准分，利用率每下降5%，该项评分减一个单位的基准分。

f)所设计硬件加速器方案最适用场景及不适用场景（具体的稀疏特性，矩阵尺寸等）分别是什么？最适用场景和不适用场景的处理。
