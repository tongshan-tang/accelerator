# PROJECT_PLAN：创芯大赛稀疏矩阵运算双架构预研计划书

## 1. 简短说明
**项目目的**：在 Xilinx Kintex-7 平台上构建高性能的大尺寸稀疏矩阵运算（乘法、加法、减法）硬件加速系统，挑战处理带宽 >75GB/s、硬件逻辑利用率极高且全局翻转率 <60% 的极限瓶颈指标。

**推进策略**：由于极高内存带宽吞吐与极致低功耗在微架构上往往存在设计制衡，我们在数据通路搭建初期采用“A/B 测试”双架构并行预研策略。我们将面向不同的内存排布格式，分别对核心算力和调度机制进行建模，而后通过布局布线预综合（OOC Synthesis）给出的绝对 PPA（Power, Performance, Area）数值，来指导我们淘汰劣体，最终收敛至少量、高效的统一架构。

## 2. 核心微架构备选方案 (A/B Testing)

### 方案 A：打包存储与行展开架构 (AoS - Array of Structures)
**体系本质**：用内存空间和宽总线换取控制流的极简，追求在最少时钟周期内达到最大局部数据吞吐，消除写回气泡。

- **数据格式**：统一的 32-bit Token 打包格式。
  - `[31:16]`：FP16 数值净数据；
  - `[15:10]`：保留位与 Valid (有效指示) 位；
  - `[9]`：EOL (End of Line) 标志位；
  - `[8:0]`：坐标索引 IDX。
  - **总线支持**：采用 256-bit 或超宽总线单周期读取 8 个 Token 对齐流水线。
- **核心机制 (行展开+直接散列)**：将运算主矩阵（例如乘法中的矩阵 A）全行依据其内部的 IDX 直接散列载入片内大深度（512 深度）的局部 LUTRAM（即 Row-Buffer）。当流矩阵（矩阵 B）按列流过时，将直接截取 B 的 IDX 映射为 A-Row-Buffer 的读地址。若抓取到 Valid 位置为 1 则送入底层运算网络；若 Valid 为 0，触发**操作数隔离 (Operand Isolation)** 将后级计算锁死。
- **优缺点分析 (Trade-offs)**：
  - **优势**：状态机极致简单，依靠物理空间直接匹配出数据重叠部分，达成单周期一拍出结果；无流水线气泡，不存在多轮重算时写回（Read-Modify-Write）的访存冲突。
  - **劣势**：高宽带总线伴随 32-bit Token 中的无效数据同样进入读写周期，动态翻转率的下压需要在门控逻辑上倾注大量心血，若不能精准切割使能，极易造成动态功耗超标。

### 方案 B：解耦读写与预匹配机制 (SoA - Structure of Arrays)
**体系本质**：时间与读总线解耦，用提前期的预匹配和精准投递换取总线上每一比特翻转的有意义化，冲击极低功耗。

- **数据格式**：数据与索引在物理 SRAM 隔离存储。
  - **Index_SRAM**：专门存储结构骨架，主要包含 9-bit IDX 位与 1-bit EOL 标志位；
  - **Data_SRAM**：纯净阵列，仅保存 16-bit FP16 数值区。
- **核心机制 (索引先行流水线)**：架构分为超宽解算级与数据投递级两级。
  - **Look-ahead 流水线**：前级调度使用超宽数据总线高速预读出两条向量的索引阵列（一拍可读 16+ 个）。经过专门且独立的匹配引擎并行比较后，打出一套“命中掩码 (Match Mask)”，将 Mask 置入异步弹性 FIFO 缓冲区。
  - **按需静默与提取**：后置数据投递级通过窥探 FIFO 传来的 Mask 掩码，只有在需要交算时，才精准向对应的 `Data_SRAM` 通道置高读使能信号。
- **优缺点分析 (Trade-offs)**：
  - **优势**：跨越连续零元素大片段的能力极强；对于平均 <30% 的赛题要求，未命中期间 Data_SRAM 可保持绝对静默与极低的读出活动数，功耗指标能够直逼理论下限。
  - **劣势**：前级海量比较器和流水线对齐机制不仅严重推高了寄存器与 LUT 开销，而且复杂的读后对齐控制网络很容易吃满底层逻辑资源，给 <75% 资源占比的要求带来极大压力。

## 4. 任务分工
确保架构隔离、基准模型对齐、前端与后沿各司其职，坚决摒弃灰色地带。本部分的职责划分贯穿于整个项目的全生命周期。

### 成员 1：软件侧与算法主责
**执行核心：** 对全系统 FP16 浮点算法精确度负责，提供无差别黄金验证模型。
- **前期探路**：剖析 `CASE/` 目录下的测试集数据，打通 `Index` 与 `Matrix` 文本到 Python 对象的通路。
- **基准模型**：引入 numpy 硬件级截断、舍入模式，产生绝对正确的黄金测试结果，并完成 A/B Testing 软件模型。
- **全周期支持**：负责后续所有增量测试用例的生成，协同 RTL 侧在比对出错时进行波形反解和逻辑纠偏，保证软硬件算法同构。

### 成员 2：RTL 设计与后端主责
**执行核心：** 从架构预研到最终实现，对硬件带宽、利用率与 PPA 极限指标负责。
- **架构初探**：前期协助分析非零元素碰撞与稀疏率，从理论带宽视角完善推演并确定最终使用【方案 A】或【方案 B】。
- **代码实现**：主导核心模块（mac_core, pe_unit, matcher等）的 Verilog 编码，以及控制核心与各 SRAM/片内缓存的高效互联。
- **后端收敛**：推进综合与布局布线，解决 Setup/Hold 时序违例，进行门控时钟等保守的微架构低功耗调优，确保最终全局翻转率达标。

### 成员 3：工程实现与验证主责
**执行核心：** 测试平台搭建、覆盖率收敛及板级验证。
- **基础验证**：前期整理验证边界，规划自动化的数据注入流程与自检验证框架。
- **动态仿真**：编写高覆盖率的 Testbench，针对流水线气泡、连续反压、操作数隔离失效等角落场景进行回归测试，确保功能零缺陷。
- **系统集成**：负责后续上板调试准备，包括但不限于 FPGA 引脚约束、ILA 探针抓捕、以及在真实硬件平台上的极限吞吐实测。
